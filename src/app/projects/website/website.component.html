Ontdekken

Als laatste activiteit heb ik gekozen om een eigen website te ontwikkelen die later kan dienen als visitekaartje bij sollicitatiegesprekken. Omdat Angular mij bijzonder heeft geboeid in het eerste trimester van het schooljaar en omdat mijn stage voornamelijk bestond uit programmeren in Angular wou ik mij graag verdiepen in wat dit framework nog meer te bieden heeft.

Ik heb gebruik gemaakt van Angular Material, een UI-componentenbibliotheek voor het bouwen van aantrekkelijke, consistente en functionele webpagina's. In de eerste plaats heb ik de documentatie van Angular Material doorgenomen en gekeken welke componenten ik kon gebruiken voor mijn website. De input componenten en cards wou ik zeker implementeren.

Daarnaast heb ik ook gebruik gemaakt van Tailwindcss voor een custom design te creëren. In tegenstelling tot Bootstrap komt Tailwindcss niet met voorgedefinieerde componenten maar voorziet het een set van CSS helper klasses. Deze klasses kunnen eenvoudig en snel gebruikt worden doorheen het project. Via het YouTube kanaal Tailwind Labs heb ik veel inzichten gekregen in de mogelijkheden die het framework te bieden heeft.

Mijn doel bij het ontwikkelen van deze website is om zoveel mogelijk kennis rond Angular, Angular Material en Tailwindcss op te doen voornamelijk omdat ik deze kant uit wil in de toekomst. Ik heb bij het maken extra aandacht besteed om de website zo gebruiksvriendelijk en intuïtief mogelijk te maken. Wat de layout betreft wou ik zoveel mogelijk verschillende klasses van Tailwind uitproberen. Ik wist op voorhand dat ik nooit hetzelfde niveau van website zou kunnen bereiken dan wanneer ik deze website zou maken met behulp van een content management systeem en kant-en-klare sjablonen. Het hoofddoel hier was niet om de mooiste website te maken maar eerder zoveel mogelijk kennis rond bovenstaande technologieën op te doen.

Uitvoeren

Eens ik door alle documentatie ben gegaan was het tijd om aan de slag te gaan. Volgende stappen heb ik doorlopen om tot het eindresultaat te komen.

  1. Algemene structuur van website

  2. Structuur in code opzetten (modules, componenten, routering)

  3. Tekst invoegen

  4. UI schikken m.b.v. Tailwindcss

  5. Hosting

Ik ben begonnen met het uitdenken van hoe de website er structureel moest uitzien. Hierbij denk ik aan de verschillende pagina’s die ik wou opnemen, welke content waar moest komen… Vervolgens ben ik deze structuur in code gaan opzetten. Hiermee bedoel ik meer bepaald het opdelen van de website in verschillende modules, componenten en de routering die hierbij hoort. Een module is een soort van container voor samenhangende code die de applicatie in kleinere onderdelen verdeelt. Ik wou werken met modules omdat deze zorgen voor een betere performantie doordat ze lazy loading mogelijk maken waardoor een module enkel laadt wanneer deze nodig is. Vervolgens heb ik al mijn geschreven content op de juiste plaats ingevoegd. Daarna was het tijd voor de UI onder handen te nemen. Dit ging relatief vlot omdat Tailwind zeer gemakkelijk en aangenaam is om te gebruiken.

De laatste stap was het hosten van de website. Aangezien de afstudeerrichting van Switch2IT software development is, zijn wij relatief weinig in contact gekomen met het deployen en hosten van applicaties. Daarom was het even zoeken naar hoe ik dit het best zou aanpakken. Ik heb verschillende hosting diensten onderzocht van AWS, Microsoft Azure tot Firebase. Ik heb voor Firebase gekozen omdat we hier tijdens de opleiding mee in aanraking zijn gekomen. Na wat opzoekwerk en enkele YouTube tutorials is het mij gelukt om mijn website te hosten.

Reflecteren

Tijdens mijn stage heb ik uitgebreid kennis kunnen maken met Angular. Daarom leek Angular mij het aangewezen framework voor de ontwikkeling van mijn eigen website. De uitdaging hier was om van nul af aan een project op te zetten, uit te werken en te hosten zonder enige begeleiding. Dit project was als het ware een test in hoe verre ik Angular onder de knie heb. In retrospectie vond ik het hele project redelijk vlot gaan. Ik durf zeggen dat ik over een uitgebreide kennis van Angular beschik en ik kan gemakkelijk mijn weg vinden binnen het framework.

Verder heb ik gekozen om Angular Material en Tailwindcss te implementeren. Met Angular Material was ik eveneens tijdens mijn stage in contact gekomen. Tailwindcss was echter nieuw en dit framework heb ik leren kennen via een Pluralsight video. Ik was hier nieuwsgierig naar en dit leek mij geschikt voor snel en eenvoudig de css van mijn website te structureren. Tailwind heeft mij meer inzichten gegeven in hoe de css achterliggend werkt. Ik vond Tailwind zeer aangenaam om mee te werken, goede documentatie en duidelijke voorbeelden zijn online te vinden.

Wat ik vooral een groot voordeel vind van Angular is dat het framework verschillende handvaten biedt om gestructureerd te werken. Ook de documentatie van Angular is echt een meerwaarde. Wat Tailwind betreft ben ik aangenaam verrast hoe snel en gemakkelijk het is om de UI aan te passen. Het nadeel is wel dat het nog relatief nieuw is en bij problemen weinig oplossingen online te vinden zijn.

Van alle projecten die ik heb gedaan, vond ik deze het leukst om te doen en heb ik het gevoel dat ik hier het meest aan ga hebben naar de toekomst toe.
